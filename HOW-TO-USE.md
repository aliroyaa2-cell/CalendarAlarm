# 📱 دليل بناء التطبيق على GitHub — خطوة بخطوة

> هذا الدليل لمن لم يستخدم GitHub قبل. اتبع الخطوات بالترتيب.
>
> **المدة المتوقعة:** 15-25 دقيقة (أول مرة)، ثم 5 دقائق للتحديثات اللاحقة.

---

## 🎯 الخلاصة قبل البدء

GitHub مثل "مصنع تطبيقات" مجاني. ترفع كود المشروع عليه، وهو يبني لك APK جاهز تلقائياً. لا تحتاج Android Studio ولا أي شيء معقد.

العملية باختصار:
```
ترفع المشروع على GitHub → ينبني تلقائياً → تنزّل APK → تثبّته
```

## المرحلة 1: إنشاء حساب GitHub
## ━━━━━━━━━━━━━━━━━━━━━━━━━━

> **تخطّى هذه المرحلة لو عندك حساب GitHub.**

### الخطوات:

1. افتح متصفح Chrome أو Safari في جهازك (لابتوب أفضل)
2. اذهب إلى: **https://github.com/signup**
3. أدخل بريدك الإلكتروني → اضغط Continue
4. أنشئ كلمة مرور قوية (15 حرف فأكثر) → Continue
5. اختر اسم مستخدم (سيظهر في رابط مشاريعك مثل `github.com/username`)
6. سيرسل GitHub رمز تحقق على بريدك → أدخل الرمز
7. اختر الخطة المجانية (**Free**) — تكفي تماماً

✔️ **الحساب جاهز.**

## المرحلة 2: إنشاء مستودع (Repository)
## ━━━━━━━━━━━━━━━━━━━━━━━━━━

> "Repository" يعني مجلد المشروع على GitHub.

### الخطوات:

1. بعد تسجيل الدخول، اذهب إلى: **https://github.com/new**

2. **Repository name:** اكتب `CalendarAlarm` (بدون مسافات)

3. **Description (اختياري):** اكتب مثلاً "تطبيق منبه التقويم"

4. اختر **Public** (مجاني — البناء يعمل بلا حدود)
   > ⚠️ لا تختر Private إلا لو تبغى المشروع سري — في الـ Free plan فيه قيود على البناء للمشاريع Private

5. **لا تضع علامة على أي من هذه:**
   - ❌ Add a README file
   - ❌ Add .gitignore
   - ❌ Choose a license

   (لأنها كلها موجودة في المشروع المرفق)

6. اضغط الزر الأخضر **Create repository**

✔️ **المستودع جاهز — لكنه فاضي. الآن سنرفع الملفات.**

## المرحلة 3: رفع ملفات المشروع
## ━━━━━━━━━━━━━━━━━━━━━━━━━━

> هذي أهم مرحلة. خذ وقتك.

### قبل البدء:
- فك ضغط ملف `CalendarAlarm2-fixed.zip` على جهازك
- ستجد مجلد اسمه `CalendarAlarm2-fixed/` فيه الملفات

### الخطوات:

1. في صفحة المستودع الجديد على GitHub، ستجد رسالة:

   *"Quick setup — if you've done this kind of thing before"*

   تحتها رابط: **"uploading an existing file"** — اضغط عليه

2. ستفتح صفحة رفع الملفات. الآن:

   **افتح المجلد `CalendarAlarm2-fixed/` على جهازك**

3. اختر **محتويات المجلد كلها** (Ctrl+A على Windows أو Cmd+A على Mac)

   ⚠️ **مهم جداً:** اسحب **محتويات** المجلد، **ليس** المجلد نفسه!

   الملفات والمجلدات التي يجب أن تختارها:
   ```
   ✔️ app/
   ✔️ gradle/
   ✔️ .github/
   ✔️ build.gradle
   ✔️ gradle.properties
   ✔️ settings.gradle
   ✔️ .gitignore
   ✔️ CHANGES.md
   ✔️ HOW-TO-USE.md
   ```

4. **اسحبها كلها** إلى منطقة الرفع في صفحة GitHub (المنطقة المنقطة)

5. سترى الملفات تظهر في صفحة GitHub. **انتظر حتى يكتمل الرفع** (شريط أزرق يكتمل)

   مدة الرفع: 1-2 دقيقة حسب الإنترنت.

6. في الأسفل، ستجد خانتين:
   - **Commit changes** الكبيرة: اكتب فيها `Initial upload`
   - الخانة الصغيرة: اتركها فارغة

7. تأكد أن **"Commit directly to the main branch"** مختار

8. اضغط الزر الأخضر **Commit changes**

✔️ **الملفات الآن على GitHub.**

### ⚠️ تنبيه للمتصفحات:

في **Safari (Mac/iPhone)**: قد لا يظهر خيار رفع المجلدات. لو واجهت مشكلة، استخدم **Chrome** أو **Firefox** بدلاً منه.

في **Edge (Windows)**: يعمل بشكل ممتاز عادةً.

## المرحلة 4: متابعة البناء التلقائي
## ━━━━━━━━━━━━━━━━━━━━━━━━━━

> البناء يبدأ تلقائياً فور رفع الملفات. لا تحتاج تشغيله يدوياً.

### الخطوات:

1. في صفحة المستودع، انظر للأعلى ستجد تبويبات:
   ```
   Code | Issues | Pull requests | Actions | Projects | ...
   ```

2. اضغط **Actions** (في الأعلى)

3. ستجد بناءً قيد التشغيل:
   - 🟡 **دائرة صفراء دوّارة** = البناء يشتغل الآن
   - ✅ **دائرة خضراء** = البناء نجح
   - ❌ **دائرة حمراء** = البناء فشل

4. اضغط على البناء (السطر العلوي اللي يقول "Initial upload")

5. اضغط على **build** (تحت Jobs على اليسار)

6. ستشاهد خطوات البناء تتم خطوة خطوة:
   ```
   ✓ Checkout code
   ✓ Set up JDK 17
   ✓ Setup Gradle 8.9
   ✓ Generate gradle wrapper if missing
   ✓ Cache Gradle dependencies
   ⏳ Build Debug APK         ← هذي تأخذ 3-5 دقائق
   ⏳ Get APK info
   ⏳ Upload APK artifact
   ⏳ Create Release
   ```

7. **انتظر 5-10 دقائق** (المرة الأولى تأخذ وقت أطول، المرات التالية أسرع)

8. عندما تتحول كل الخطوات لـ ✅ خضراء → البناء انتهى بنجاح ✔️

## المرحلة 5: تنزيل الـ APK
## ━━━━━━━━━━━━━━━━━━━━━━━━━━

> فيه طريقتين — اختر الأسهل لك.

### 🟢 الطريقة 1: من Releases (الأسهل)

1. ارجع لصفحة المستودع الرئيسية (اضغط على اسم المشروع في الأعلى)

2. على اليمين، ستجد قسم **Releases** — اضغط عليه

3. ستجد الإصدار الأحدث في الأعلى: `CalendarAlarm Build 1`

4. تحت **Assets**، اضغط على `app-debug.apk` لتنزيله

5. الملف الآن في مجلد Downloads على جهازك

### 🟡 الطريقة 2: من Artifacts

1. اذهب لتبويب **Actions**

2. اضغط على البناء الناجح (الدائرة الخضراء)

3. انزل لأسفل الصفحة → ستجد قسم **Artifacts**

4. اضغط على `CalendarAlarm-debug-1` لتنزيله

5. ستحصل على ملف ZIP — فكّ ضغطه → ستجد `app-debug.apk`

✔️ **الـ APK جاهز عندك.**

## المرحلة 6: تثبيت الـ APK على جوالك
## ━━━━━━━━━━━━━━━━━━━━━━━━━━

### 1. أرسل الـ APK لجوالك

**اختر طريقة:**
- ✔️ واتساب لنفسك (Web → Phone)
- ✔️ Google Drive
- ✔️ كابل USB
- ✔️ بلوتوث
- ✔️ Telegram لنفسك

### 2. على جوالك:

1. **احذف النسخة القديمة من التطبيق أولاً** (مهم جداً)
   - اضغط مطوّلاً على أيقونة CalendarAlarm
   - اختر "Uninstall" أو "إزالة"
   - أكّد

2. افتح ملف `app-debug.apk` على الجوال

3. ستظهر رسالة "For your security, your phone is not allowed to install unknown apps from this source"

4. اضغط **Settings**

5. فعّل **"Allow from this source"**

6. ارجع — سيبدأ التثبيت

7. اضغط **Install**

✔️ **التطبيق مثبّت.**

### 3. بعد التثبيت — افحص الإعدادات الحرجة:

افتح ملف **CHANGES.md** (داخل المجلد المضغوط) واتبع قسم "خطوات ما بعد التثبيت" — مهم جداً للسماح للتطبيق بإيقاظ الشاشة على Samsung.

## التحديثات لاحقاً (مهم اقرأها)
## ━━━━━━━━━━━━━━━━━━━━━━━━━━

### إذا أعطيتك ZIP محدّث في المستقبل:

**الطريقة الصحيحة لرفع تحديث:**

1. افتح صفحة المستودع على GitHub: `github.com/[username]/CalendarAlarm`

2. لكل ملف معدّل في الـ ZIP الجديد:
   - اضغط على الملف في GitHub
   - اضغط أيقونة **القلم** (✏️) في أعلى يمين الملف
   - الصق المحتوى الجديد كاملاً
   - في الأسفل، اضغط **Commit changes**

   **أو الأسهل — احذف المشروع كاملاً وارفعه من جديد:**

3. **الطريقة الأسرع للتحديثات الكبيرة:**
   - اذهب إلى **Settings** (آخر تبويب في المستودع)
   - انزل لآخر الصفحة → **Danger Zone** → **Delete this repository**
   - اكتب اسم المستودع للتأكيد
   - أنشئ مستودع جديد بنفس الاسم وارفع الملفات الجديدة

4. **البناء يبدأ تلقائياً** بعد رفع التحديث

5. **APK جديد** سيكون جاهز في Releases خلال 5-10 دقائق

### تشغيل بناء يدوي (بدون رفع تحديث):

1. تبويب **Actions**
2. على اليسار، اضغط **Build APK**
3. على اليمين، اضغط **Run workflow** (الزر الرمادي)
4. اختر الـ branch: **main**
5. اضغط الزر الأخضر **Run workflow**

## استكشاف الأخطاء
## ━━━━━━━━━━━━━━━━━━━━━━━━━━

### ❌ "البناء فشل" (دائرة حمراء)

1. اضغط على البناء الفاشل
2. اضغط على **build** (تحت Jobs)
3. ابحث عن السطر الذي به ❌ حمراء
4. اضغط عليه ليُفتح
5. **انسخ الخطأ كاملاً** والصقه لي في المحادثة

⚠️ **لا تجرّب حلولاً عشوائية.** أرسل لي الخطأ وسأشخّصه في رد واحد.

### ❌ "Installation blocked" (التثبيت محجوب)

**على Samsung Galaxy:**
1. **Settings → Biometrics and security → Install unknown apps**
2. اختر التطبيق اللي فتح ملف الـ APK (Files أو My Files)
3. فعّل **"Allow from this source"**

**على جوالات أخرى:**
- Settings → Apps → Special access → Install unknown apps

### ❌ "App not installed"

السبب الأشهر: نسخة قديمة من التطبيق مثبّتة بـ signature مختلف.

**الحل:** احذف النسخة القديمة كاملاً قبل التثبيت.

### ❌ "البناء يأخذ أكثر من 30 دقيقة"

مستحيل — أوقفه واعد التشغيل:
1. تبويب **Actions** → اضغط البناء
2. اضغط **Cancel workflow** (في الأعلى يمين)
3. شغّل بناء يدوي جديد

### ❌ "Releases فاضية"

تأكد من:
1. البناء انتهى بنجاح (✅ خضراء)
2. أعطيت الـ workflow صلاحية: تبويب **Settings → Actions → General** → **Workflow permissions** → اختر **"Read and write permissions"** → احفظ

## أسئلة شائعة
## ━━━━━━━━━━━━━━━━━━━━━━━━━━

### كم مرة أقدر أبني في اليوم؟

**Public repository:** بلا حدود ✔️
**Private repository:** 2000 دقيقة شهرياً (~133 بناء شهرياً)

### هل GitHub مجاني فعلاً؟

نعم، 100% مجاني للاستخدامك. لا توجد رسوم خفية.

### هل APK يحتاج تجديد؟

لا. APK ثابت — يشتغل ما دام الجوال يدعمه.

### هل يقدر شخص ثاني يستخدم نفس المشروع؟

نعم — Public repository يعني أي شخص يقدر يبني نفس التطبيق لنفسه. لكن APK نفسه ما ينحط على Play Store — هو للاستخدام الشخصي فقط.

### هل أقدر أحذف المشروع لو ندمت؟

نعم، أي وقت — Settings → Danger Zone → Delete repository.

## 📌 الخلاصة في 5 سطور

1. **رفع المشروع على GitHub** = `Create repo → Upload files → Commit`
2. **GitHub Actions يبني تلقائياً** = ينتظر بدون فعل أي شيء
3. **الـ APK جاهز في Releases** = حمّله ولصقه على الجوال
4. **التحديث = نفس العملية** = رفع الملفات الجديدة → APK جديد تلقائياً
5. **أي خطأ → أرسله لي** = حل واحد مؤكد، بدون تجارب
